Filesystem (C++17)

Introduction

So, as it can be known from the title, it was first introduced in C++17. This came with many new and simplified functions and operations which reduced the many lines of code that were necessary in the previous versions of C++ (A comparison between the programs in C++11 and using C++17's filesystem is elaborated in Carlos's Filesystem in C++17). For a while, filesystem was not a standard C++ header. Hence, many compilers or text editors (usually the older versions) may not be able to execute the program succesfully or may require a different header file (which is discussed next).

Compatibility and Compilation

Did you ever stumble across two different kinds of headers (include <experimental/filesystem> and include <filesystem>) and wondered why there were two? Well, here's the answer:
(Don't worry if you haven't though. You will know all about it now:)

Back when filesystem was still a pre-standard header, compilers like GCC v6 and v7 supported the "Filesystem Technical Specification" which was defined in the <experimental/filesystem> header. After it was given the status of standardized C++ header, the compilers since then supported the <filesystem> header.

So, now that we know the difference between the headers, let's go to the compilation!

Firstly, the compiler needs to know that we are using C++17. We can do so using -c++1z after calling the compiler. Secondly, with the pre-standard versions, -lstdc++fs is added at the end of the command making the compilation command look like:
g++ -std=c++1z main.cpp -lstdc++fs
This is also supported clang++.

GCC v8 and higher versions support the <filesystem> implementation with the -std=c++17 flag.

Functions

Now that we know how to compile our program and check if our functions are all right, let's move to the most interesting part: Filesystem Functions!

Note:

We need to call the functions using the std::experimental::filesystem (for older versions of compiler that support pre-standard filesystem header) and std::filesystem (for those supporting the standard filesystem header). So in order to reduce our efforts, we define it as:
namespace fs = std::experimental::filesystem
or
namespace fs = std::filesystem
